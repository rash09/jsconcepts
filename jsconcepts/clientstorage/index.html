Client Storage
Local Storage

LocalStorage is a Storage object, it inherits from the StoragePrototype that has methods and property:

  • setItem
  • getItem
  • removeItem
  • length
  • key
  • clear

Local storage blocks the DOM on browsers (Chrome, Firefox, Edge) while writing data.
In Safari browser, large data update might lead to permanent browser freeze

Session Storage

Data stored in sessionStorage have an expiration time. this means that sessionStorage data is cleared when the page session ends. A page session is held during the lifetime of a tab. , it inherits from the StoragePrototype that has methods and property:

  • setItem
  • getItem
  • removeItem
  • length
  • key
  • clear

Local storage blocks the DOM on browsers (Chrome, Firefox, Edge) while writing data.
In Safari browser, large data update might lead to permanent browser freeze

IndexedDB

IndexedDB formerly known as WebSimpleDB the Indexed Database API and uses the concept of an object store for client side data. A asynchronous API.

In Firefox & Chrome, IndexedDB is much slower than Localstorage for insertion operation.
Works fine in Safari & Edge without any interuption.
Safari doesnt support IndexedDB inside a web worker.
